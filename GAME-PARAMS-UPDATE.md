## 🔄 更新内容

### 修改前的问题

**错误提取**：
- ❌ 分数（score）- 这是动态值，不是配置参数
- ❌ 当前得分 - 游戏过程中会变化

**用户反馈**：
> "分数"这个不属于参数，参数应该是生命数、生命值、地图尺寸等

---

### 修改后的规则

**正确提取**：
- ✅ **生命数/生命值** - 游戏机制参数（初始值，不会变）
- ✅ **地图尺寸** - 游戏场景大小（固定配置）
- ✅ **速度** - 玩家/敌人移动速度
- ✅ **大小** - 角色/道具尺寸
- ✅ **生成率** - 敌人生成间隔

## 📊 参数分类详解

### 1. ❤️ 生命/血量（最高优先级）

**提取模式**：
```javascript
// 生命数（整数）
lives = 3
const maxLives = 5
playerLives = 3

// 生命值/血量（可以是小数）
hp = 100
const maxHealth = 100.0
player.health = 50
```

**示例游戏**：
- 平台跳跃游戏 - 生命数
- RPG 游戏 - 血量值
- 射击游戏 - 生命值

**不属于参数**：
- ❌ 当前得分（动态变化）
- ❌ 当前血量（动态变化）
- ✅ 最大生命值（配置参数）

## ❌ 不提取的参数

### 动态值（不是配置参数）

| 参数 | 原因 | 示例 |
|------|------|------|
| **当前分数** | 游戏过程中变化 | `score = 0`（初始值可以提取，但不作为关键参数） |
| **当前血量** | 游戏过程中变化 | `currentHP = 50` |
| **当前关卡** | 游戏过程中变化 | `currentLevel = 1` |
| **游戏时间** | 游戏过程中变化 | `timeElapsed = 0` |
| **击杀数** | 游戏过程中变化 | `killCount = 0` |

### 为什么不提取分数？

**原因**：
1. 分数是**动态累积值**，不是游戏机制配置
2. 每次游戏开始都重置为 0
3. 调整分数没有实际意义（玩家关心的是最高分）

**应该提取**：
- ✅ 目标分数（winScore, targetScore）- 胜利条件
- ✅ 最高分（highScore）- 保存在 localStorage
