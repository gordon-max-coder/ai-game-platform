# 🎮 GameAI - 我的游戏功能使用说明

## ✅ 已完成的功能

### 1. 模块化架构
项目采用模块化设计，代码健壮、易维护：

```
ai-game-platform/
├── js/
│   ├── game-storage.js      # 游戏存储管理模块
│   ├── game-ui.js           # 游戏 UI 渲染模块
│   ├── my-games-page.js     # 我的游戏页面逻辑
│   └── create-save.js       # 创建页面保存逻辑
├── css/
│   └── my-games.css         # 我的游戏页面样式
├── my-games.html            # 我的游戏页面
├── create.html              # 创建游戏页面（已集成保存功能）
└── 我的游戏 - 使用说明.md    # 本文档
```

### 2. 核心功能

#### ✅ 自动保存
- 每次生成新游戏时自动保存到"我的游戏"
- 每次修改游戏后自动更新保存
- 无需手动操作

#### ✅ 游戏管理
- 查看所有已创建的游戏
- 统计信息（游戏数、版本数、代码量）
- 删除不需要的游戏

#### ✅ 二次创作
- 点击"✏️ 修改"按钮进入编辑模式
- 在原有游戏基础上继续优化
- 版本自动递增

#### ✅ 直接游玩
- 点击"▶️ 开始游戏"立即游玩
- 在新窗口打开，全屏体验
- 支持多次打开

#### ✅ 数据导出/导入
- 导出所有游戏数据为 JSON 文件
- 从备份文件导入游戏
- 支持跨设备迁移

### 3. 使用流程

#### 创建游戏
1. 访问 `http://localhost:3000/create.html`
2. 输入游戏描述
3. 点击"生成游戏"
4. ✅ **自动保存到"我的游戏"**

#### 查看游戏
1. 访问 `http://localhost:3000/my-games.html`
2. 查看所有已创建的游戏
3. 查看统计信息

#### 修改游戏
1. 在"我的游戏"页面找到要修改的游戏
2. 点击"✏️ 修改"按钮
3. 自动跳转到创建页面并加载原游戏
4. 输入修改需求（如：增加武器、添加 Boss）
5. 点击"生成修改版本"
6. ✅ **自动更新到"我的游戏"**

#### 删除游戏
1. 在"我的游戏"页面找到要删除的游戏
2. 点击"🗑️ 删除"按钮
3. 确认删除

#### 导出数据
1. 访问"我的游戏"页面
2. 点击"📤 导出数据"按钮
3. 下载 JSON 文件到本地

#### 导入数据
1. 访问"我的游戏"页面
2. 点击"📥 导入数据"按钮
3. 选择之前导出的 JSON 文件
4. 确认导入

### 4. 数据存储说明

#### 存储位置
- 浏览器的 localStorage
- 键名：`gameai_my_games`
- 数据格式：JSON 数组

#### 数据结构
```json
{
  "id": "game_1234567890_abc",
  "code": "<!DOCTYPE html>...",
  "codeLength": 25000,
  "title": "星际战士",
  "description": "在浩瀚宇宙中战斗...",
  "type": "动作",
  "style": "科幻",
  "prompt": "创建一个太空射击游戏...",
  "version": 3,
  "createdAt": "2025-03-06T12:00:00.000Z",
  "updatedAt": "2025-03-06T14:00:00.000Z"
}
```

#### 存储限制
- 浏览器 localStorage 通常为 5-10MB
- 每个游戏约 20-50KB
- 可存储约 100-200 个游戏
- 超出限制时会提示清理旧游戏

### 7. 常见问题

#### Q: 游戏保存后在哪里？
A: 访问 `http://localhost:3000/my-games.html` 查看所有游戏

#### Q: 如何继续修改游戏？
A: 在"我的游戏"页面点击"✏️ 修改"按钮

#### Q: 删除的游戏能恢复吗？
A: 不能，删除操作不可逆，请谨慎操作

#### Q: 数据会丢失吗？
A: 
- 只要不清除浏览器数据，游戏会一直保存
- 建议定期使用"导出数据"功能备份
- 清除浏览器缓存会删除所有游戏

#### Q: 可以在另一台电脑上玩吗？
A: 
- 使用"导出数据"功能导出游戏
- 在另一台电脑上使用"导入数据"功能导入
- 或直接复制 localStorage 数据

#### Q: 最多能保存多少个游戏？
A: 
- 取决于浏览器 localStorage 限制（通常 5-10MB）
- 每个游戏约 20-50KB
- 建议保留 20-50 个常用游戏

### 8. 技术细节

#### 模块依赖
```
my-games.html
├── game-storage.js (核心存储)
├── game-ui.js (UI 渲染)
└── my-games-page.js (页面逻辑)

create.html
├── game-storage.js (核心存储)
└── create-save.js (保存逻辑)
```

#### 主要 API
```javascript
// 保存游戏
GameStorage.saveGame(gameData);

// 更新游戏
GameStorage.updateGame(gameId, updates);

// 删除游戏
GameStorage.deleteGame(gameId);

// 获取所有游戏
GameStorage.getAllGames();

// 获取统计
GameStorage.getStats();

// 导出数据
GameStorage.exportData();

// 导入数据
GameStorage.importData(jsonString);
```
