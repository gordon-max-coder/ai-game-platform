# 📝 提示词优化速查卡

> 让 Gemini 生成 Claude 级别代码的秘诀

---

## 🎯 一句话总结

**具体 > 模糊，量化 > 描述，结构 > 随意**

---

## 📊 对比示例

### ❌ 差提示词 (生成粗糙代码)

```
"做一个贪食蛇游戏，要好看一点"
```

**问题**:
- ❌ 太模糊，模型需要猜测
- ❌ 没有具体参数
- ❌ 没有视觉规范
- ❌ 没有代码要求

**预期结果**: 能运行但粗糙，100-200 行基础代码

---

### ✅ 好提示词 (生成优质代码)

```
创建完整贪食蛇游戏，HTML5 Canvas 实现。

【核心功能】
- 蛇：绿色渐变 (#2ecc71→#27ae60)，圆角方块，带眼睛
- 食物：红色圆形 (#e74c3c)，脉动动画
- 速度：初始 8 格/秒，每吃 5 个 +1，最高 15 格/秒
- 计分：当前分 + 最高分 (localStorage)
- 碰撞：撞墙/撞自己游戏结束

【视觉效果】
- 画布：600x600，深色背景 (#1a1a2e)
- 网格：淡灰细线 (#333355)，间距 20px
- 特效：粒子爆炸 (吃食物)，屏幕闪烁 (游戏结束)

【游戏状态】
- 开始界面：标题 + "按空格开始"
- 游戏结束：显示分数 + "按 R 重新开始"
- 暂停：按 P 键

【代码要求】
- 单文件 HTML，IIFE 模式
- 函数：init(), gameLoop(), draw(), update()
- 使用 requestAnimationFrame
- 详细注释，400-600 行

输出完整代码。
```

**优点**:
- ✅ 具体参数 (速度、颜色、尺寸)
- ✅ 视觉规范 (色值、特效)
- ✅ 完整流程 (开始/进行/结束)
- ✅ 代码要求 (结构、注释)

**预期结果**: 精美完整，400-600 行优质代码

## 🔑 6 大优化原则

### 1. 量化参数

| ❌ 模糊 | ✅ 量化 |
|--------|--------|
| "移动快一点" | "速度：8 格/秒" |
| "大一点" | "尺寸：600x600" |
| "颜色深一点" | "颜色：#1a1a2e" |
| "多几分" | "每个食物：+10 分" |

---

### 2. 定义颜色

| ❌ 模糊 | ✅ 具体 |
|--------|--------|
| "红色" | "#e74c3c" |
| "深色背景" | "背景：#1a1a2e" |
| "绿色蛇" | "蛇头：#2ecc71，渐变到 #27ae60" |
| "好看的颜色" | 提供完整色板 |

---

### 3. 描述特效

| ❌ 模糊 | ✅ 具体 |
|--------|--------|
| "加特效" | "粒子爆炸：5-8 个碎片散开" |
| "动画效果" | "食物脉动：缩放 10%，0.5 秒周期" |
| " transition" | "淡入淡出：300ms" |

---

### 4. 指定结构

| ❌ 模糊 | ✅ 具体 |
|--------|--------|
| "代码整洁" | "IIFE 模式，无全局变量" |
| "函数清晰" | "init(), gameLoop(), draw(), update()" |
| "定义常量" | "GRID_SIZE=20, CANVAS_SIZE=600" |

---

### 5. 完整流程

| ❌ 只有游戏 | ✅ 完整体验 |
|-----------|-----------|
| 直接开始 | 开始界面 → 说明 → 开始 |
| 突然结束 | 结束界面 → 分数 → 重开 |
| 无法暂停 | P 键暂停/继续 |
| 没有说明 | 操作说明 + 规则 |

---

### 6. 列出"不要"

```
【避免的问题】
- 不要用外部库 (纯原生 JS)
- 不要用 setInterval (用 requestAnimationFrame)
- 不要全局变量污染
- 不要硬编码魔法数字
- 不要省略注释
```

## 💡 进阶技巧

### 1. 分步生成

```
第 1 步：先做基础框架 (能运行)
第 2 步：添加核心功能 ( playable)
第 3 步：优化视觉效果 (变美)
第 4 步：添加额外功能 (音效、存档)
```

```

```


## 🎯 总结

### 核心心法

**把模型当成实习生**：
- 你给的任务越具体，它做的越好
- 你说"好看点"，它不知道什么是好看
- 你说"用#2ecc71 绿色"，它就知道怎么做

### 三个转变

1. **从"做什么" → "怎么做"**
   - ❌ "做个贪食蛇"
   - ✅ "蛇用#2ecc71 绿色，20x20px，8 格/秒..."

2. **从"要好看" → "要什么颜色"**
   - ❌ "好看的颜色"
   - ✅ "背景#1a1a2e，蛇#2ecc71，食物#e74c3c"

3. **从"代码整洁" → "这样组织"**
   - ❌ "代码要好"
   - ✅ "用 IIFE，函数有 init(), gameLoop()..."
