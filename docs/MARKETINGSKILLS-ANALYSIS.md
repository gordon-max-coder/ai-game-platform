## 📋 实际包含的功能

根据 SKILL.md 分析，`marketingskills` 技能包含以下**核心能力**：

### ✅ 已实现的功能

#### 1. **A/B 测试设计** (核心功能)
- 假设框架设计
- 测试类型选择（A/B、A/B/n、MVT、Split URL）
- 变量设计（标题、文案、视觉、CTA、内容）
- 流量分配策略

#### 2. **样本量计算**
- 基于基线转化率和预期提升
- 提供快速参考表
- 集成 Evan Miller's 和 Optimizely 计算器
- 详细指南：`references/sample-size-guide.md`

#### 3. **指标选择**
- 主要指标（Primary Metric）
- 次要指标（Secondary Metrics）
- 护栏指标（Guardrail Metrics）
- 指标框架和选择原则

#### 4. **测试实施指导**
- 客户端实施（JavaScript）
- 服务端实施
- 工具推荐（PostHog、Optimizely、VWO 等）

#### 5. **测试运行管理**
- 预启动检查清单
- 测试期间注意事项
- 避免提前停止（Peeking Problem）

#### 6. **结果分析**
- 统计显著性检验（95% 置信度）
- 效应大小评估
- 细分分析
- 结果解释框架

#### 7. **文档模板**
- 测试记录模板
- 假设文档模板
- 结果报告模板
- 位置：`references/test-templates.md`

#### 8. **最佳实践和常见错误**
- 测试设计原则
- 执行注意事项
- 分析陷阱避免

## 💡 GameAI 项目应用建议

### 当前可以做的（仅使用 marketingskills）

#### 1. 创作页面 A/B 测试
```markdown
假设：因为用户经常不知道创建什么游戏（从空白输入框推断），
我们 believe 添加"🎲 灵感"按钮 
will cause 创作率提升 30%
for 首次访问用户。
We'll know this is true when 创作完成率从 20% 提升到 26%。
```

**样本量计算**：
- 基线：20%
- 预期提升：30%（相对提升，即 20% → 26%）
- 每组需要：约 1,500 用户
- 测试时长：根据日活计算

**指标**：
- 主要：创作完成率（点击"生成"按钮/访问页面）
- 次要：平均输入长度、灵感按钮点击率
- 护栏：页面停留时间、跳出率

#### 2. 游戏命名测试
```markdown
假设：因为具体名称比通用名称更有吸引力，
我们 believe 从 prompt 提取具体关键词（如"贪食蛇"）
will cause 游戏点击率提升 20%
for 游戏库访问者。
```

#### 3. 三栏布局测试
```markdown
假设：因为参数可视化让用户更容易理解游戏，
we believe 显示右侧参数面板
will cause 修改率提升 25%
for 编辑模式用户。
```

### 需要额外技能才能做的

1. **实施追踪**（需要 analytics-tracking）
   - 设置"创作完成"事件
   - 追踪"灵感按钮"点击
   - 计算转化率

2. **生成变体文案**（需要 copywriting）
   - "创建游戏" vs "开始创作" vs "生成我的游戏"
   - 不同的输入框占位符
   - 不同的按钮文案

3. **CRO 建议**（需要 page-cro）
   - 创作页面布局优化
   - 减少摩擦点
   - 增加社会证明
