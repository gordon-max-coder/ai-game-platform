# 🎮 通用游戏生成提示词模板

> 适用于任何类型的游戏，让 Gemini 生成 Claude 级别的代码

---

## 📋 万能公式

```
[游戏名称] + [技术栈] + [核心功能清单] + [视觉规范] + [代码要求] + [用户体验] + [输出格式]
```

## 🎮 常见游戏类型模板

### 打砖块游戏

```
请创建一个完整的打砖块游戏，使用 HTML5 Canvas 和原生 JavaScript。

【技术要求】
- 单文件 HTML
- Canvas API 渲染
- 画布 800x600
- 代码模块化，有注释

【核心功能】

1. 挡板控制
   - 鼠标或左右箭头键控制
   - 挡板宽度：100px，高度：15px
   - 移动范围：画布底部左右
   - 颜色：蓝色渐变 (#3498db → #2980b9)

2. 球体系统
   - 初始位置：挡板上方中心
   - 半径：8px，白色
   - 初始速度：5 像素/帧
   - 发射角度：随机 30-60 度
   - 碰到挡板反弹 (根据碰撞点改变角度)

3. 砖块系统
   - 5 行 x 10 列 = 50 块砖
   - 每块尺寸：70x20px，间距 5px
   - 每行不同颜色 (彩虹色)
   - 被球击中后消失
   - 全部消除后胜利

4. 计分系统
   - 每块砖：10 分
   - 显示当前分数
   - 最高分 (localStorage)
   - 连击奖励 (连续击中 + 额外分)

5. 生命系统
   - 初始 3 条生命
   - 球掉落底部：失去 1 生命
   - 生命耗尽：游戏结束
   - 显示剩余生命 (心形图标)

6. 游戏状态
   - 开始界面：标题、说明、"点击开始"
   - 游戏进行：正常游戏
   - 暂停：按 P 键
   - 胜利：显示"恭喜获胜!"
   - 失败：显示"游戏结束"

【视觉效果】

1. 背景
   - 深色渐变 (#0c0c1e → #1a1a3e)
   - 星空效果 (随机小白点)

2. 特效
   - 球撞砖：粒子爆炸 (5-8 个碎片)
   - 球撞挡板：闪光效果
   - 砖块消失：淡出动画

3. UI
   - 分数：左上角，白色 24px
   - 生命：右上角，红色心形
   - 暂停提示：中央半透明 overlay

【难度系统】

- 初始球速：5 像素/帧
- 每消除 10 块砖：速度 +0.5
- 最高速度：10 像素/帧
- 显示"加速!"提示

【代码要求】

- IIFE 模式，无全局污染
- 函数：init(), gameLoop(), draw(), update(), spawnParticles()
- 常量：CANVAS_SIZE, PADDLE_SIZE, BALL_SPEED, COLORS
- requestAnimationFrame 渲染
- 详细注释

【输出】
完整单文件 HTML，预计 500-700 行。
```

### 飞机大战游戏

```
请创建一个完整的飞机大战游戏，使用 HTML5 Canvas。

【核心功能】

1. 玩家飞机
   - 位于画布底部中央
   - 鼠标或键盘控制移动
   - 自动连续射击
   - 3 个等级 (普通→双发→三发)

2. 敌机系统
   - 3 种类型：
     * 小型：快速，1 点生命，红色
     * 中型：中速，3 点生命，黄色
     * 大型：慢速，10 点生命，Boss 级
   - 从顶部随机出现
   - 向下移动

3. 子弹系统
   - 玩家子弹：向上，蓝色
   - 敌机子弹：向下，红色 (大型敌机发射)
   - 碰撞检测：子弹 vs 敌机，子弹 vs 玩家

4. 道具系统
   - 武器升级：红色"P"字
   - 生命恢复：绿色"+"字
   - 护盾：蓝色"○"
   - 炸弹：橙色"💣"

5. 计分系统
   - 击落敌机：小型 100 分，中型 300 分，大型 1000 分
   - 显示当前分数和最高分

6. 波次系统
   - 每 1000 分进入下一波
   - 敌机数量和速度递增
   - 每 3 波出现 Boss

【视觉效果】

- 背景：深空 (#0a0a2e)，滚动星空
- 玩家飞机：三角形，青色，带引擎火焰
- 敌机：不同形状的三角形/圆形
- 爆炸：粒子效果 (10-20 个碎片)
- 激光：发光效果

【代码结构】

- 类：Player, Enemy, Bullet, Particle, Game
- 方法：update(), draw(), checkCollision()
- 常量定义清晰
- 完整注释

输出完整 HTML 代码，600-800 行。
```

### 平台跳跃游戏

```
请创建一个简单的平台跳跃游戏，类似超级玛丽。

【核心功能】

1. 玩家角色
   - 可左右移动 (A/D 或箭头键)
   - 跳跃 (空格键，支持长按跳更高)
   - 二段跳能力
   - 重力加速度：0.5 像素/帧²

2. 平台系统
   - 5-8 个不同高度的平台
   - 可从下方穿过，从上方站立
   - 部分平台会移动
   - 部分平台会消失 (踩上去 2 秒后消失)

3. 收集物

   - 钻石：蓝色菱形，+50 分 (稀有)

4. 敌人
   - 巡逻怪：在平台上左右移动
   - 掉落怪：从上方掉落
   - 踩头可消灭，碰到身体死亡

5. 关卡系统
   - 3 个关卡
   - 每关有终点旗帜
   - 到达终点进入下一关

6. 生命系统
   - 初始 3 条命
   - 碰敌人或掉坑死亡
   - 显示剩余生命

【物理系统】

- 重力：0.5 像素/帧²
- 跳跃力：-12 像素/帧
- 移动速度：5 像素/帧
- 摩擦力：0.8
- 碰撞检测：AABB

【视觉效果】

- 卷轴背景 (随玩家移动)
- 玩家：方形角色，有眼睛和嘴巴
- 平台：土黄色，有草皮顶部
- 敌人：红色圆形，有角
- 金币：旋转动画

输出完整代码，700-900 行。
```

### 2048 游戏

```
请创建一个完整的 2048 游戏。

【核心功能】

1. 网格系统
   - 4x4 网格
   - 每个格子可有一个数字方块
   - 空白格子为灰色背景

2. 方块生成
   - 每次移动后随机生成
   - 90% 概率生成 2，10% 概率生成 4
   - 生成位置：随机空白格

3. 移动合并
   - 上下左右四个方向
   - 相同数字碰撞时合并
   - 合并后数字翻倍
   - 一次移动只能合并一次

4. 计分系统
   - 合并得到新数字：+新数字分数
   - 显示当前分数和最高分
   - 动画显示得分

5. 胜负判定
   - 出现 2048：胜利 (可继续玩)
   - 无法移动：游戏结束

6. 游戏控制
   - 箭头键或 WASD 移动
   - 空格键重新开始
   - U 键撤销上一步

【视觉效果】

1. 方块颜色
   - 2: #eee4da
   - 4: #ede0c8
   - 8: #f2b179
   - 16: #f59563
   - 32: #f67c5f
   - 64: #f65e3b
   - 128+: #edcf72 (带发光)

2. 动画
   - 新生成：从小到大弹出
   - 合并：放大闪烁
   - 移动：平滑过渡 (200ms)

3. UI
   - 游戏标题：2048 大字
   - 分数板：圆角矩形
   - 按钮：新游戏、说明

【代码要求】

- 清晰的网格数据结构
- 分离的逻辑和渲染
- 完整的注释
- 支持撤销功能

输出完整 HTML，400-500 行。
```

## 💡 提示词优化技巧

### 1. 量化一切

**❌ 模糊**:
```
"蛇移动要快"
```

**✅ 量化**:
```
"初始速度：8 格/秒，每吃 5 个食物 +1，最高 15 格/秒"
```

### 2. 定义颜色

**❌ 模糊**:
```
"绿色的蛇"
```

**✅ 具体**:
```
"蛇头：#2ecc71，蛇身渐变到 #27ae60"
```

### 3. 指定结构

**❌ 模糊**:
```
"代码要整洁"
```

**✅ 具体**:
```
"使用 IIFE 模式，函数：init(), gameLoop(), draw(), update()"
"常量：GRID_SIZE, CANVAS_SIZE, COLORS"
```

### 4. 描述特效

**❌ 模糊**:
```
"添加特效"
```

**✅ 具体**:
```
"吃到食物时：5-8 个粒子爆炸，向四周散开，2 秒后消失"
"游戏结束时：屏幕闪烁红色 3 次，每次 0.3 秒"
```

### 5. 列出"不要"

```
【避免的问题】
- 不要使用外部库
- 不要用 setInterval
- 不要全局变量
- 不要硬编码数字
- 不要省略注释
```

## 📊 提示词长度 vs 质量

| 提示词长度 | 生成质量 | 适用场景 |
|-----------|---------|---------|
| 1-2 句 | ⭐⭐ | 快速测试 |
| 1 段 (100 字) | ⭐⭐⭐ | 简单游戏 |
| 结构化 (300-500 字) | ⭐⭐⭐⭐ | 日常使用 |
| 超详细 (800+ 字) | ⭐⭐⭐⭐⭐ | 商业项目 |

**建议**: 日常使用 **结构化版本** (300-500 字)，性价比最高

- [ ] 游戏名称和类型
