## 🚀 快速测试（3 分钟）

### 步骤 1: 打开测试页面
```
双击：test-code-modifier.html
或访问：http://localhost:3000/test-code-modifier.html
```

**预期**: 自动运行 10 个测试用例，全部通过 ✅

---

### 步骤 2: 测试真实编辑器
```
1. 访问：http://localhost:3000/create.html
2. 输入："创建一个贪食蛇游戏"
3. 点击"创建"
4. 等待游戏生成完成
```

---

### 步骤 3: 打开编辑器
```
1. 点击"🎮 编辑器"按钮
2. 看到三栏布局:
   - 左侧：组件树（Player, Food 等）
   - 中间：画布 + 工具栏
   - 右侧：属性面板
```

**预期**: 编辑器正常打开，能看到组件树

---

### 步骤 4: 测试安全修改
```
1. 点击"Player"对象
2. 找到"speed"属性
3. 修改值：300 → 500
4. 观察:
   ✅ Toast 提示"已修改 speed: 300 → 500"
   ✅ 撤销按钮变为"↩️ 撤销 (1)"
   ✅ 代码编辑器中代码已更新
```

---

### 步骤 5: 验证准确性
```
1. 在代码编辑器中搜索"speed"
2. 检查:
   ✅ 只有"const speed = 500;"被修改
   ✅ "playerSpeed"、"maxSpeed"等不变
   ✅ 代码格式保持原样
```

---

### 步骤 6: 测试撤销
```
1. 点击"↩️ 撤销 (1)"
2. 观察:
   ✅ Toast 提示"已撤销 speed 的修改"
   ✅ 代码恢复为"const speed = 300;"
   ✅ 撤销按钮变灰"↩️ 撤销"
```

---

### 步骤 7: 测试多次修改
```
1. 修改 speed: 300 → 400
2. 修改 color: "#ff0000" → "#0000ff"
3. 修改 size: 32 → 48
4. 观察撤销按钮:
   ✅ "↩️ 撤销 (3)"
5. 连续点击撤销 3 次
   ✅ 依次恢复 size → color → speed
```

---

### 步骤 8: 测试错误处理
```
1. 在属性面板找到不存在的属性（如"nonexistent"）
2. 尝试修改（需要手动输入到输入框）
3. 观察:
   ❌ Toast 提示"未找到变量 'nonexistent' 的声明"
   ✅ 没有错误崩溃
```
