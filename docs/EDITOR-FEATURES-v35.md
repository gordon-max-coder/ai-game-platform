## ✨ 当前功能

### 1. 智能对象解析

编辑器会从 AI 生成的 HTML5 代码中**自动识别**游戏对象：

```javascript
// 识别的代码模式
speed = 300          → Player.speed
color = "#ff0000"    → Player.color  
size = 32            → Player.size
x = 180, y = 320     → Player.position
```

**支持的游戏类型**:
- ✅ 贪食蛇（识别"蛇"、"贪食蛇"关键词）
- ✅ 飞机大战（识别"飞机"、"玩家"关键词）
- ✅ 打砖块（识别"砖块"、"敌人"关键词）
- ✅ 其他 Canvas 游戏

### 2. 属性实时修改

**支持的属性类型**:

| 属性 | 修改方式 | 效果 |
|------|---------|------|
| **speed** | 滑块/数字输入 | 修改游戏速度 |
| **color** | 颜色选择器 | 修改角色颜色 |
| **size** | 滑块/数字输入 | 修改角色大小 |
| **position** | X/Y 输入框 | 修改初始位置 |

**修改流程**:
```
1. 用户在属性面板修改值
   ↓
2. updateProperty() 更新内存
   ↓
3. syncPropertyToCode() 修改源代码
   ↓
4. 更新代码编辑器显示
   ↓
5. 如果正在运行，通过 postMessage 发送到 iframe
   ↓
6. 游戏实时更新
```

## 🔧 工作原理

### 代码解析示例

**原始代码**:
```javascript
const speed = 300;
const color = "#ff0000";
const size = 32;
```

**用户修改 speed 为 500**:

**编辑器操作**:
```javascript
// 1. 正则匹配
const speedRegex = /(?:speed|velocity|速度)\s*[:=]\s*\d+/gi;

// 2. 替换值
code = code.replace(speedRegex, `speed = 500`);

// 3. 结果
const speed = 500;  // ← 已修改
const color = "#ff0000";
const size = 32;
```

### 实时预览（运行中）

如果游戏正在运行，编辑器会发送 `postMessage`:

```javascript
iframe.contentWindow.postMessage({
    type: 'UPDATE_PROPERTY',
    property: 'speed',
    value: 500
}, '*');
```

**游戏需要监听消息**:
```javascript
// 游戏代码中需要添加
window.addEventListener('message', (e) => {
    if (e.data.type === 'UPDATE_PROPERTY') {
        if (e.data.property === 'speed') {
            speed = e.data.value;
        }
    }
});
```

## ⚠️ 当前限制

### 1. 代码解析限制

**能识别的模式**:
```javascript
✅ speed = 300
✅ const speed = 300
✅ var speed = 300
✅ speed: 300
✅ 速度 = 300
```

**不能识别的模式**:
```javascript
❌ const SPEED = 300  // 大写常量
❌ let speed = 300    // let 声明（部分情况）
❌ config.speed = 300 // 对象属性
❌ speeds[0] = 300    // 数组元素
```

### 2. 实时预览限制

**需要游戏支持 postMessage**:

目前 AI 生成的游戏**不支持**实时属性更新，因为游戏代码中没有监听 `postMessage`。

**解决方案**:
- 修改后点击"✅ 应用代码"重新加载
- 或者停止后重新点击"▶️ 运行"

### 3. 属性类型限制

**当前支持**:
- ✅ 数值属性（speed, size）
- ✅ 颜色属性（color）
- ✅ 位置属性（position x/y）

**暂不支持**:
- ❌ 布尔属性（visible, enabled）
- ❌ 字符串属性（name, text）
- ❌ 复杂对象（velocity 向量）

## 🐛 已知问题

### 问题 1: 修改后游戏没变化

**原因**: 游戏没有重新加载

**解决**:
```
1. 点击 "⏹️ 停止"
2. 点击 "▶️ 运行" 重新加载
或者
1. 修改属性后自动更新了代码
2. 在代码编辑器中按 Ctrl+A 全选
3. 点击 "✅ 应用" 重新加载
```

### 问题 2: 找不到要修改的属性

**原因**: 代码解析失败

**解决**:
```
1. 检查代码中变量命名
   - 使用 speed 而不是 SPEED
   - 使用 color 而不是 fillColor
   
2. 手动在代码编辑器中修改
   - 找到对应变量
   - 直接修改值
   - 点击 "✅ 应用"
```

### 问题 3: 颜色修改不生效

**原因**: 颜色格式不匹配

**解决**:
```
1. 确保使用十六进制颜色
   ✅ #ff0000
   ❌ red
   ❌ rgb(255,0,0)
   
2. 在代码中查找颜色定义
   - 可能使用了不同的变量名
   - 手动修改并应用
```

## 📝 技术细节

### 正则表达式模式

```javascript
// 速度匹配
/speed|velocity|速度\s*[:=]\s*\d+/gi

// 颜色匹配
/color|fillStyle|颜色\s*[:=]\s*["']?#[0-9a-fA-F]{6}["']?/gi

// 大小匹配
/size|radius|width|height|大小\s*[:=]\s*\d+/gi

// 位置匹配
/x|posX\s*[:=]\s*\d+/gi
/y|posY\s*[:=]\s*\d+/gi
```

### postMessage 协议

```javascript
// 编辑器发送
{
    type: 'UPDATE_PROPERTY',
    objectId: 'player',
    property: 'speed',
    value: 500
}

// 游戏接收（需要添加）
window.addEventListener('message', (e) => {
    if (e.data.type === 'UPDATE_PROPERTY') {
        // 处理属性更新
    }
});
```
