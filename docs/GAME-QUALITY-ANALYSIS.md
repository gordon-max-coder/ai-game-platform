### 第 3 轮：修复 Bug

**需求**: 修复 Bug，无法获取新子弹

**代码质量**:
```javascript
// ✅ 优点
- 尝试修复子弹系统
- 添加了弹药箱机制

// ❌ 缺点
- 代码混乱，有大量注释说明
- 有未完成的代码块

- 存在逻辑错误
- 代码量激增至 650 行
```

**严重问题**:

#### 1. 函数重复定义 ❌
```javascript
// 原始版本
function updatePlayer() { ... }
function drawUI() { ... }
function handleCollisions() { ... }

// 修正版本
function updatePlayerCorrected() { ... }
function drawUICorrected() { ... }
function handleCollisionsCorrected() { ... }

// 最后替换
updatePlayer = updatePlayerCorrected;
drawUI = drawUICorrected;
handleCollisions = handleCollisionsCorrected;
```

**问题**: 这种模式非常混乱，应该直接修改原函数！

#### 2. 未完成的代码块 ❌
```javascript
// 在 handleCollisions() 中
for (let k = 0; k < bulletsToAdd; k++) {
    // 假装添加子弹，实际上是增加了可发射次数。
    // 如果子弹是消耗品，当 player.bullets.length < player.maxBullets 时，玩家可以发射。
    // 拾取弹药箱，意味着玩家可以多发射几发。
    // 这里我们只需要确保 player.bullets.length 不会超过 player.maxBullets
    // 但由于 player.bullets 是已发射的子弹，这里的逻辑需要调整。
    // 正确的逻辑应该是：玩家有一个 `currentAmmo` 变量，表示当前可发射的子弹数量。
    // `player.bullets` 列表只存储屏幕上正在飞行的子弹。
    // 让我们修改玩家的子弹系统。

    // 临时修正：直接增加 player.maxBullets，让玩家能发射更多子弹。
    // 更合理的修正：引入 player.currentAmmo 变量。
    // player.maxBullets += ammoRestoreAmount; // 暂时这样处理
    // 为了符合"修复 Bug，无法获取新子弹"，我们应该让玩家的子弹数量（可发射次数）增加。
}
```

**问题**: 这是 AI 的思考过程，不应该出现在最终代码中！

#### 3. 逻辑错误 ❌
```javascript

particles.splice(i, 1); // ❌ 错误！应该是 powerUps.splice(i, 1);

// 问题 2: 自动射击逻辑混乱
if (!isTouching && player.dx === 0 && player.dy === 0 && ...) {
    shootBullet(); // ❌ 站着不动就自动射击？
}
```

#### 4. 代码冗余 ❌
```javascript

    x: canvas.width / 2 - 25,

    ...
};

// 后面又补充定义

## 🎯 核心问题诊断

### 问题 1: 上下文丢失 ⚠️

**现象**:
- 第 3 轮没有保留第 2 轮的完整功能
- 丢失了追踪子弹的完整逻辑
- 精英敌人的血条显示被移除

**原因**:
```
对话历史没有正确传递 → AI 看不到之前的完整代码 → 重新生成时遗漏功能
```

### 问题 2: 模型能力限制 ⚠️

**Gemini 2.5 Flash 的特点**:
- ✅ 快速、便宜
- ✅ 简单任务表现良好
- ❌ 复杂迭代容易出错
- ❌ 代码审查能力弱
- ❌ 容易引入新 Bug

**对比 Claude**:
```
Gemini 2.5 Flash:
- 第 1 轮：✅ 良好
- 第 2 轮：✅ 良好（功能增加）
- 第 3 轮：❌ 混乱（引入 Bug）

Claude Sonnet 4.6:
- 第 1 轮：✅ 优秀
- 第 2 轮：✅ 优秀（结构清晰）
- 第 3 轮：✅ 优秀（修复 Bug 不引入新问题）
```

### 问题 3: 提示词不明确 ⚠️

**当前提示词**:
```

**问题**:

**方案**:
```javascript
// 生成后自动检查
function validateCode(code) {
    // 检查是否有注释残留
    if (code.includes('// 临时修正')) return false;
    if (code.includes('// TODO')) return false;
    
    // 检查是否有未完成代码
    if (code.includes('...')) return false;
    
    // 检查函数重复定义
    const functionNames = code.match(/function (\w+)\(/g);
    const duplicates = findDuplicates(functionNames);
    if (duplicates.length > 0) return false;
    
    return true;
}
```
