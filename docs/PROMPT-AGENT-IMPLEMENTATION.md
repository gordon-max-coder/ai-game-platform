## 🎯 核心功能

### 1. 智能引导触发

**自动检测简单输入**：
- 提示词长度 < 15 字
- 细节描述 < 2 个

**示例**：
```
用户：贪食蛇
智能体：检测到简单需求，启动引导 ✅

用户：创建一个贪食蛇游戏，蛇速 8 像素/帧，有障碍物和道具
智能体：已有足够细节，直接生成 ✅
```

### 2. 游戏类型识别

支持 6 种游戏类型：

| 类型 | 关键词 | 问题数 |
|------|--------|--------|
| 🐍 贪食蛇 | 贪食蛇、蛇蛇、snake | 4 |
| 🧱 打砖块 | 打砖块、砖块、breakout | 4 |
| 🚀 射击游戏 | 射击、飞机、太空 | 4 |
| 🏃 平台跳跃 | 跳跃、平台、冒险 | 4 |
| 🧩 益智游戏 | 拼图、消除、2048 | 3 |
| 🏃 跑酷游戏 | 跑酷、奔跑、地铁 | 4 |

### 3. 多题型支持

**单选题**：
```
🎨 你希望游戏是什么视觉风格？
🌈 鲜艳多彩  ⚪ 极简风格  👾 像素复古  💡 霓虹灯
```

**滑块题**：
```

[====●=====]  7 - 🚶 正常
```

**多选题**：
```
✨ 需要哪些额外功能？（可多选）
✅ 💊 道具系统  ✅ ✨ 粒子特效
```

### 4. 智能提示词生成

根据用户选择生成详细提示词：

**输出示例**：
```
创建一个贪食蛇游戏。蛇的移动速度为 7 像素/帧。
初始长度为 5，每吃一个食物增长 1 节。

吃食物和死亡时有粒子爆炸效果。

代码结构清晰，使用 IIFE 避免全局变量污染。
包含完整的游戏循环（开始、运行、结束）。
有分数显示和游戏结束界面。
支持键盘和触摸控制。
添加粒子效果和动画增强视觉体验。
使用鲜艳的渐变色彩，视觉效果丰富。
```

## 🔧 技术实现

### 核心类结构

```javascript
class PromptAgent {
    // 游戏类型配置
    gameTypes = { snake, breakout, shooter, ... }
    
    // 状态管理
    state = {
        isActive: false,
        gameType: null,
        step: 0,
        answers: {},
        basePrompt: ''
    }
    
    // 主要方法
    detectGameType(prompt)      // 识别游戏类型
    needsGuidance(prompt)       // 判断是否需要引导
    startGuidance(prompt)       // 启动引导
    getNextQuestion()           // 获取下一题
    recordAnswer(id, answer)    // 记录答案
    generateFinalPrompt()       // 生成最终提示词
    reset()                     // 重置状态
}
```

### 集成方式

```javascript
// 在 create-new.js 中
async function generateGame() {
    const prompt = elements.promptInput.value;
    
    // 检查是否需要智能体引导
    if (!currentGameId && promptAgent && !agentGuidanceActive) {
        const needsGuidance = promptAgent.needsGuidance(prompt);
        
        if (needsGuidance) {
            const guidanceResult = promptAgent.startGuidance(prompt);
            
            if (guidanceResult) {
                startAgentGuidance(prompt);
                return;  // 暂停，等待用户完成引导
            }
        }
    }
    
    // 继续正常生成流程...
}
```

## 💡 最佳实践

### ✅ 推荐

1. **简短输入触发引导** - <15 字自动引导
2. **问题数量 3-4 个** - 避免用户疲劳
3. **图标 + 文字** - 提升可读性
4. **进度条显示** - 让用户知道进度
5. **支持跳过** - 高级用户可跳过

### ❌ 避免

1. **问题太多** - 超过 5 题用户烦躁
2. **专业术语** - 使用通俗语言
3. **强制引导** - 始终提供跳过选项
4. **复杂选项** - 每问题不超过 5 选项

## 🐛 已知问题

### 1. 浏览器缓存

**问题**: 修改后仍显示旧版本

**解决**: 强制刷新（Ctrl+Shift+R）或更新版本号
