# 🔧 解决 CORS 问题 - 使用代理服务器

## 问题原因

浏览器出于安全考虑，阻止了从本地文件（`file://`）直接调用外部 API（`https://jiekou.ai`）。

错误信息：
```
Access to fetch at 'https://jiekou.ai/api/chat/completions' from origin 'null' 
has been blocked by CORS policy
```

## 解决方案

使用本地代理服务器转发 API 请求。

---

## 🚀 快速开始

### 步骤 1: 安装 Node.js

如果还没有安装 Node.js，请下载安装：
https://nodejs.org/

安装后验证：
```bash
node --version
```

### 步骤 2: 启动代理服务器

**方法 A: 双击启动（推荐）**

双击文件：`start-proxy.bat`

**方法 B: 命令行启动**

```bash
cd C:\Users\jiangym\.copaw\ai-game-platform
node proxy-server.js
```

你会看到：
```
🚀 代理服务器启动中...
📡 监听端口：3000
🌐 API 地址：https://jiekou.ai

✅ 服务器已启动
🔗 本地地址：http://localhost:3000
🔗 代理端点：http://localhost:3000/proxy/chat/completions
```

### 步骤 3: 打开创建页面

**保持代理服务器窗口运行**，然后：

1. 打开浏览器
2. 访问：`create.html`
3. 刷新页面（F5）
4. 现在可以正常生成游戏了！

## 🔍 工作原理

```
浏览器 (create.html)
    ↓
http://localhost:3000/proxy/chat/completions
    ↓
代理服务器 (proxy-server.js)
    ↓
https://jiekou.ai/api/chat/completions
    ↓
返回响应
```

代理服务器的作用：
- ✅ 绕过 CORS 限制
- ✅ 转发请求到真实 API
- ✅ 返回响应给浏览器

## 🐛 故障排除

### 问题 1: 端口被占用

**错误**: `Error: listen EADDRINUSE: address already in use :::3000`

**解决**:
```bash
# 方法 1: 关闭占用端口的程序
# 方法 2: 修改代理端口（编辑 proxy-server.js，改 PORT 为其他值）
```

### 问题 2: Node.js 未安装

**错误**: `'node' is not recognized...`

**解决**:
1. 安装 Node.js: https://nodejs.org/
2. 重启命令行
3. 验证：`node --version`

### 问题 3: 代理服务器无法启动

**检查**:
- [ ] Node.js 已安装
- [ ] 端口 3000 未被占用
- [ ] 防火墙允许本地连接

### 问题 4: 仍然提示 CORS 错误

**解决**:
1. 确认代理服务器正在运行
2. 刷新浏览器页面（F5）
3. 检查浏览器控制台，确认请求地址是 `http://localhost:3000/proxy/...`

## 🔒 安全说明

### 本地开发使用

此代理服务器仅用于**本地开发测试**：
- ✅ 只在本地运行（localhost）
- ✅ 不存储任何数据
- ✅ 不修改请求内容
- ⚠️ **不要部署到公网**

### API 密钥安全

API 密钥存储在前端代码中（`js/create-direct.js`），仅用于本地测试。

生产环境应该：
1. 将密钥放在服务器端
2. 使用环境变量
3. 添加认证机制
