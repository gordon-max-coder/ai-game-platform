# 🎯 模型选择快速指南

> 为什么 Gemini 生成的游戏比较粗糙？

---

## 📊 一句话答案

**决定游戏质量的关键**：
1. ⭐⭐⭐⭐⭐ **模型架构和训练数据** (最重要)
2. ⭐⭐⭐⭐⭐ **推理能力和代码理解**
3. ⭐⭐⭐⭐ **指令遵循能力**
4. ⭐⭐⭐ 最大输出 tokens
5. ⭐⭐ 上下文长度 (影响最小)

**Gemini Flash Lite 质量差的原因**：
- ❌ 它是**轻量级模型** (经济舱 vs 头等舱)
- ❌ 参数可能被**压缩/蒸馏**
- ❌ 定位是**快速便宜**而非高质量

## 🚀 提升 Gemini 质量的技巧

### 1. 超详细 Prompt
```
❌ "做贪食蛇"
✅ "创建完整贪食蛇游戏，要求：
    - 蛇身绿色渐变，头部更深
    - 红色圆形食物
    - 计分系统和最高分
    - 碰撞检测和结束动画
    - 深色背景 + 网格线
    - 开始界面和重新开始功能"
```

### 2. 分步生成
1. 基础框架 → 2. 游戏逻辑 → 3. 视觉效果 → 4. 额外功能

### 3. 提供示例
```
"参考这个代码风格：[粘贴优质代码]
请用类似方式实现..."
```

### 4. 调整参数
```javascript
temperature: 0.5,  // 降低创造性，提高稳定性
max_tokens: 8000   // 确保足够输出空间
```

## 🎓 总结

### 为什么 Gemini 质量差？
- ❌ **不是**上下文不够 (1M 远超需求)
- ❌ **不是**输出限制 (65K 完全足够)
- ✅ **而是**模型定位 (轻量级 = 经济舱)
- ✅ **而是**训练深度 (通用 vs 代码优化)

### 关键结论
1. **模型架构 > 参数大小** - 旗舰模型就是更好
2. **代码训练 > 通用能力** - 专项优化很重要
3. **详细 Prompt > 模型切换** - 好的指令提升质量
4. **混合使用 > 单一模型** - 性价比最高
