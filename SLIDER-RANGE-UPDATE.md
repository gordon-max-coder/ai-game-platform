## 🎯 需求说明

**用户反馈**：
> 游戏数值的初始值和范围要合理，默认数值是当前数值。
> 范围：0 - 默认值*2
> 例如生命初始值是 5，滑动条就停在 5 的位置，然后范围就是 0-10。

---

## 🔄 修改内容

### 修改前

**范围计算**：
```javascript
const min = Math.max(0, Math.floor(value * 0.1));  // 最小值：value 的 10%
const max = Math.ceil(value * 2);                   // 最大值：value 的 2 倍
```

**问题**：
- ❌ 最小值不是 0，调整不便
- ❌ 范围不够直观
- ❌ 默认值位置不明确

**示例**（生命值=5）：
```
范围：1 - 10
滑块位置：5
用户：为什么最小值是 1 不是 0？
```

---

### 修改后

**范围计算**：
```javascript
const min = 0;                              // 最小值：固定为 0
const max = Math.ceil(value * 2);           // 最大值：value 的 2 倍
const step = value < 1 ? 0.1 : 0.5 : 1;     // 步长：自动调整
```

**改进**：
- ✅ 最小值固定为 0，符合直觉
- ✅ 范围清晰：0 到 默认值×2
- ✅ 默认值在中间位置，可增可减
- ✅ 显示默认值提示

**示例**（生命值=5）：
```
范围：0 - 10（默认值：5）
滑块位置：5（正中间）
用户：可以调到 0（无敌模式）或 10（困难模式）
```

## 📝 代码变更

### 修改的文件

**文件**: `js/game-analyzer.js`

**修改位置**: `render()` 函数中的滑块生成部分

**修改前**：
```javascript
const min = Math.max(0, Math.floor(value * 0.1));
const max = Math.ceil(value * 2);
const step = value < 1 ? 0.1 : (value < 10 ? 0.5 : 1);

html += `
    <div class="property-hint">范围：${min} - ${max}</div>
`;
```

**修改后**：
```javascript
const min = 0;
const max = Math.ceil(value * 2);
const step = value < 1 ? 0.1 : (value < 10 ? 0.5 : 1);

html += `
    <div class="property-hint">范围：${min} - ${max}（默认值：${value}）</div>
`;
```

**变更**：
1. ✅ 最小值固定为 0
2. ✅ 提示中添加默认值显示

## 📊 数值范围规则

### 自动步长规则

| 数值范围 | 步长 | 示例 |
|---------|------|------|
| value < 1 | 0.1 | 0.5 → 步长 0.1 |
| 1 ≤ value < 10 | 0.5 | 5 → 步长 0.5 |
| value ≥ 10 | 1 | 100 → 步长 1 |

### 范围计算规则

```
最小值 = 0
最大值 = ceil(默认值 × 2)
默认位置 = 默认值
```

**示例**：
- 生命=3 → 范围 0-6，步长 0.5
- 速度=7 → 范围 0-14，步长 0.5
